## 安全

微软高度重视软件产品和服务的安全性，这包括通过 GitHub 组织托管的所有源代码仓库：[Microsoft](https://github.com/Microsoft)、[Azure](https://github.com/Azure)、[DotNet](https://github.com/dotnet)、[AspNet](https://github.com/aspnet)、[Xamarin](https://github.com/xamarin) 等。

如果你认为在任何微软仓库中发现了符合[微软安全漏洞定义](https://aka.ms/opensource/security/definition)的安全问题，请按以下方式报告。

## 报告安全问题

**请不要在公开的 GitHub issue 中报告安全漏洞。**

请通过 Microsoft Security Response Center (MSRC) 提交：[https://msrc.microsoft.com/create-report](https://aka.ms/opensource/security/create-report)。

如需匿名提交，可发送邮件至 [secure@microsoft.com](mailto:secure@microsoft.com)。如可行，请使用我们的 PGP 密钥加密（[PGP 密钥页面](https://aka.ms/opensource/security/pgpkey)）。

通常会在 24 小时内收到回复。如未收到，请邮件跟进以确认我们收到了你的消息。更多信息见 [microsoft.com/msrc](https://aka.ms/opensource/security/msrc)。

请尽量提供以下信息，帮助我们更快分析：

  * 问题类型（如缓冲区溢出、SQL 注入、XSS 等）
  * 相关源文件的完整路径
  * 受影响源代码的位置（tag/branch/commit 或直接链接）
  * 复现所需的特殊配置
  * 复现步骤
  * 概念验证或利用代码（如有）
  * 影响范围，包括潜在利用方式

信息越完整，处理越高效。若提交漏洞赏金，完整报告有助于更高的奖励。详情见 [Microsoft Bug Bounty Program](https://aka.ms/opensource/security/bounty)。
